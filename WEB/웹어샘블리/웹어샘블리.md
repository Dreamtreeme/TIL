웹 애플리케이션 증가

html css javascirpt 로 사용될때는 단수 정적페이지

하지만 자바스크립트의 발전으로 극복

그러나 자바스크립트는 유연하지만 복잡한 계산이나 그래픽 처리에 한계

이런 한계를 극복하고자 웹어샘블리 등장

자바스크립트의 실행과정

js파일 -> 브라우저가 소스코드를 읽어 분석 -> 컴파일과 실행, 최적화를 함. 그러나 어려움. 데이터를 추측과 동시에 연산해야함

웹어샘블리는 파싱작업이 필요없이 바로 컴파일+추상화가 가능해서 자바스크립트보다 빠르게 실행이가능

이너리 명령어 형식:

인간이 읽을 수 있는 일반적인 텍스트 기반 코드가 아닌, 컴퓨터가 더 빠르고 효율적으로 처리할 수 있도록 설계된 낮은 수준의 바이트코드 형식입니다.

컴파일 대상:

C, C++, Rust와 같은 여러 프로그래밍 언어의 코드를 웹 브라우저에서 실행할 수 있도록 컴파일되는 이식 가능한 컴파일 대상입니다.

실행 환경:

주요 웹 브라우저에 내장되어 있어 자바스크립트와 함께 실행되며, 가상 스택 머신에서 코드를 실행합니다.

목표:

웹에서 네이티브에 가까운 성능을 제공하여, 고성능을 요구하는 게임, 비디오/이미지 편집, 과학적 시뮬레이션 등 복잡한 애플리케이션을 웹에서 구동할 수 있게 하는 것입니다.

자바스크립트와의 관계:

Wasm은 자바스크립트를 대체하는 것이 아니라, 자바스크립트가 처리하기 어려웠던 성능 집약적인 작업을 맡아 보완하는 역할을 합니다. Wasm 모듈은 자바스크립트 API를 통해 호출되고 상호작용합니다.

대부분의 브라우저에서 지원함.

하지만 자바스크립트는 쉽고 유연. 이런 개발의 유용점을 완전히 버릴 필요는 없음.

특정 경우에는 웹 어샘블리가 필요함

1. 대량의 데이터처리
2. 실시간 미디어 처리
3. 복잡한 수학 연산
